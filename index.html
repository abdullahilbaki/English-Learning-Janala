Frequently Asked Questions

1. What is the difference between var, let, and const?

var is the old way of declaring variables. It’s function-scoped and can be re-declared. It also gets hoisted, which can cause weird bugs.

let is block-scoped (only exists within {}). It can be updated but not re-declared in the same scope. Safer than var.

const is also block-scoped, but can’t be reassigned after it's set. Useful for values that shouldn’t change.

2. What is the difference between map(), forEach() , and filter()?

All three are used to loop through arrays, but they work differently:

map() Creates a new array by applying a function to each element. Great for transformations.

forEach() Just loops through the array and runs a function on each item. Doesn’t return anything.

filter() Creates a new array with only the elements that match a condition.

3. Explain arrow functions and how they are different from regular functions?

Arrow functions are a more concise way to write functions in JavaScript. They use the => syntax, making them shorter and easier to read. Unlike regular functions, arrow functions do not have their own this ; instead, they inherit this from their surrounding scope. This makes them especially useful in situations where we want to avoid the confusion of this changing inside callbacks or event handlers.

Another key difference is that arrow functions do not have their own arguments object. In regular functions, arguments contains all the parameters passed to the function, even if they are not explicitly declared. However, arrow functions do not have this feature, so if we need a similar functionality, we must use rest parameters instead.

Arrow functions also cannot be used as constructor functions. In JavaScript, regular functions can be used with the new keyword to create object instances, but arrow functions do not have this capability. If we try to use new with an arrow function, it will result in an error.

Overall, arrow functions are best used when we need a shorter syntax and do not require this, arguments, or constructor functionality. They work well for simple functions, especially in callbacks, array methods like map() and filter() , and when we want to maintain the context of this, in a function. Regular functions, on the other hand, are more suitable when we need access to this, arguments, or need to create objects using new.

4. how JavaScript Promises work?

JavaScript Promises handle asynchronous operations, like fetching data or reading files, without blocking the main thread. A Promise represents a value that might be available now, later, or never.

A Promise has three states: pending (waiting), fulfilled (successful), or rejected (failed). When a Promise is fulfilled, it runs the .then() method. If it fails, the .catch() method handles errors. The .finally() method runs regardless of success or failure.

Promises help avoid callback hell, making async code cleaner and easier to manage. They are commonly used in APIs, databases, and async functions like fetch() .

5. How closures work in JavaScript?

Closures in JavaScript happen when a function remembers and keeps access to variables from its outer scope, even after the outer function has finished running.

This works because JavaScript functions create their own scope, and inner functions can "close over" variables from their parent function. This allows data to stay private and be reused later.

Closures are useful for data hiding, creating private variables, and making functions more flexible. They are commonly used in event handlers, timers, and function factories.
